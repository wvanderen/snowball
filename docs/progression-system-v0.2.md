# Snowball Progression System v0.2

## 1. Purpose

This document supersedes the early v0.1 progression notes as the current planning target for Snowball's core game systems.

The v0.2 direction is built around five clarifications:

1. Global XP is deprecated.
2. Each sphere has its own XP, level curve, and permanent identity progression.
3. Energy is the main scalable power currency and funds Core Power, upgrades, routing, glyph systems, and slots.
4. The lattice is mechanically playable: energy flows through individually controlled connections.
5. Glyphs are variable rewards with rising costs, strategic synergies, and power-scaling slots.

The daily-use promise remains unchanged:

```txt
open app -> tap sphere -> start session -> real effort feeds the lattice
```

Progression should deepen that loop without making starting harder.

---

## 2. Resource model

### Sphere XP

Sphere XP replaces global XP.

```txt
1 focused minute in a sphere = 1 XP for that sphere
```

Sphere XP represents real history inside a domain. It primarily:

- advances that sphere's level curve;
- grants sphere progression points at level thresholds;
- unlocks permanent sphere-identity upgrades;
- gates deeper tiers of that sphere's tech tree;
- eventually contributes to prestige, memory, and long-tail identity systems.

Sphere XP is not the main spendable currency. It is the record of investment that unlocks a sphere's permanent growth.

#### XP scaling stance

Baseline XP should remain grounded in real time, but late-game upgrades and glyphs may provide balanced XP multipliers or bonus XP conditions.

These should be used carefully and usually appear later in the progression curve.

Good XP modifiers:

```txt
High Momentum grants +5% Sphere XP.
First milestone completion of the day grants bonus Sphere XP.
A rare glyph grants +XP during active sessions while Momentum is above 80.
A Core Power upgrade improves XP from completed milestones, not idle time.
```

Risky XP modifiers:

```txt
Massive passive XP while away.
Multipliers that make short fake sessions optimal.
Global XP boosts that erase sphere identity.
```

Design rule:

```txt
XP boosts can accelerate practiced spheres, but should not replace practice.
```

### Sphere Level

Each sphere has a level determined by its Sphere XP.

Sphere levels grant a sphere-specific progression currency, currently called **Sphere Points**.

```txt
Sphere XP -> Sphere Level -> Sphere Points -> permanent sphere upgrades
```

Sphere Points are local to the sphere that earned them.

```txt
Music XP levels Music.
Music levels grant Music Sphere Points.
Music Sphere Points are spent only on Music upgrades.
```

### Sphere Points

Sphere Points are spent on permanent sphere-identity upgrades: the sphere's tech tree, paths, aspects, or equivalent upgrade surface.

Sphere Points are not bought with Energy. They are earned by real engagement with that sphere.

They encourage the user to actually spend time in a life domain, not only optimize the economy around it.

### Energy

Energy is the main game currency and scalable power resource.

Generated by:

- active sessions;
- passive sphere production;
- milestone Blooms;
- routing effects;
- glyph triggers;
- Charge releases;
- later combo systems.

Spent on:

- Core Power upgrades;
- sphere power upgrades;
- connection and routing upgrades;
- glyph rolls/unlocks;
- glyph slot upgrades;
- sphere slots;
- center upgrades;
- respecs and quality-of-life systems.

Energy is allowed to scale heavily over long play. Snowball can take inspiration from incremental games where late-game costs and output become very large.

### Core Power

Core Power replaces global XP / generic game level.

Core Power is the global upgrade menu and account/lattice progression spine.

It is purchased primarily with Energy through the Center Sphere.

Core Power can unlock:

- additional sphere slots;
- higher passive/active production baselines;
- connection caps;
- manual routing tools;
- glyph forge tiers;
- glyph slot caps;
- center upgrades;
- automation and presets;
- deeper upgrade tiers;
- advanced lattice mechanics.

Core Power should feel like the main global loop:

```txt
practice -> Energy -> invest in Center -> Core Power -> bigger lattice options
```

### Momentum

Momentum is local consistency health and performance intensity for a sphere.

Momentum should synergize with XP and Energy systems, but it should not become a punitive streak replacement.

Primary effects:

- Energy output multiplier;
- Bloom strength;
- routing throughput or loss reduction;
- glyph trigger conditions;
- unlock discounts or temporary attunement;
- late-game or rare XP bonuses.

Example baseline:

```txt
Energy multiplier = 0.5 + momentum / 100
```

Example late-game XP synergy:

```txt
If Momentum >= 75, focused minutes grant +10% Sphere XP.
```

## 3. Progression architecture

Snowball v0.2 has three major progression layers.

### Local sphere progression

```txt
Sphere XP -> Sphere Level -> Sphere Points -> sphere tech tree / permanent identity
```

Purpose:

- make each sphere develop from real use;
- create domain-specific mechanical identity;
- reward time spent in that sphere;
- avoid tying mechanics to the sphere's real-life label.

### Global Core progression

```txt
Energy -> Core Power -> global upgrades and capacity
```

Purpose:

- provide the main scalable power sink;
- replace global XP;
- unlock lattice-wide systems;
- make the Center Sphere meaningful.

### Variable reward progression

```txt
Energy -> glyph roll / reward -> inventory -> socket -> synergy
```

Purpose:

- create exciting reward moments;
- add build variety;
- create strategic decisions and synergies;
- provide escalating energy sinks.

---

## 4. Sphere tech tree / permanent upgrades

Sphere upgrades should use both Sphere Points and Energy, but in different roles.

Recommended model:

```txt
Sphere XP determines level.
Level grants Sphere Points.
Sphere Points unlock or purchase permanent identity nodes.
Energy pays for power, amplification, routing, slots, and other scalable upgrades.
```

A permanent sphere upgrade may require:

```txt
Required Sphere Level: 4
Required prerequisite node: Flow Rank 1
Cost: 1 Sphere Point
Optional Energy Cost: 250 Energy
```

Use Energy costs when the upgrade provides direct power scaling. Use Sphere Points when the upgrade defines identity or unlocks a new behavior.

### Upgrade categories

#### Identity upgrades

Bought mainly with Sphere Points.

Examples:

```txt
Flow Rank 1: This sphere routes outgoing Energy more efficiently.
Bloom Rank 1: Milestone completion creates a stronger Bloom.
Anchor Rank 1: Momentum decays more gently.
Charge Rank 1: This sphere stores some incoming Energy as Charge.
```

#### Power upgrades

Bought mainly with Energy.

Examples:

```txt
Increase this sphere's base output.
Increase this sphere's passive production.
Increase active-session multiplier.
Increase max Charge.
Upgrade an edge's throughput.
```

#### Hybrid upgrades

Require sphere level or Sphere Points, then cost Energy.

Examples:

```txt
Requires Flow Rank 2; costs Energy; unlocks a second outgoing route.
Requires Sphere Level 6; costs Energy; unlocks a glyph slot on this sphere.
Requires Anchor Rank 3; costs Energy; raises Momentum floor.
```

This allows both real-world effort and game-system engagement to matter.

---

## 5. Core Power menu

Core Power is purchased at the Center with Energy.

Core Power upgrades are global and should be expensive enough to feel like major milestones.

Example categories:

### Lattice capacity

- unlock additional domain sphere slots;
- increase max connections;
- unlock non-center edges;
- unlock routing allocation controls;
- unlock routing presets.

### Production scaling

- global passive baseline +x%;
- global active-session baseline +x%;
- Center conversion efficiency +x%;
- milestone Bloom baseline +x%.

### Glyph systems

- unlock Glyph Forge;
- unlock higher glyph rarities;
- unlock glyph inventory capacity;
- unlock extra glyph slot caps;
- unlock duplicate fusion or glyph leveling.

### Quality-of-life / strategic tools

- save routing layouts;
- discounted respecs;
- better post-session reward preview;
- bulk edge toggles;
- reduced routing loss near Center.

Core Power should avoid replacing sphere identity. It makes the lattice stronger and broader; sphere XP makes individual spheres deeper.

## 6. Energy routing and lattice control

Energy flow should be integrated with the lattice because the lattice is the game board.

Each connection should eventually be an individually controlled object.

```ts
type Connection = {
  id: string;
  fromSphereId: string;
  toSphereId: string;
  enabled: boolean;
  allocationPercent: number;
  level: number;
  throughputMultiplier: number;
  routingLoss: number;
  mode: ConnectionMode;
};
```

Initial connection modes:

```txt
Conduit: normal Energy transfer.
```

Future connection modes:

```txt
Valve: holds flow until threshold, then releases.
Lens: amplifies flow but adds loss.
Capacitor: converts part of flow into Charge.
Echo: sends a smaller effect to neighbors.
Gate: only opens under certain Momentum or milestone conditions.
```

### Near-term routing model

For v0.2 implementation planning, the first useful model is:

- each connection can be enabled or disabled;
- each sphere can send Energy across one or more outgoing connections;
- outgoing connections have allocation percentages;
- total outgoing allocation should normalize to 100% among enabled edges;
- if the user has not configured routing, use a safe default route to Center;
- routes should be visible as flowing lines on the lattice.

This makes each edge strategically relevant without requiring a full automation game immediately.

### Routing and Momentum

Momentum can improve routing.

Examples:

```txt
High Momentum reduces outgoing routing loss.
A Flow upgrade converts Momentum into throughput.
A glyph gives bonus output when routing through a high-Momentum sphere.
```

## 7. Glyph system v0.2

Glyphs are modular build pieces and variable rewards.

They should feel like meaningful reward moments, similar to:

- choosing a card reward after a Slay the Spire fight;
- reaching a major Cookie Clicker threshold and buying a new permanent upgrade;
- finding a synergistic item that changes the shape of a build.

Glyphs are not just a static picker. They need acquisition, rarity, escalating cost, and socket pressure.

### Glyph loop

```txt
Generate Energy -> spend Energy / complete milestone -> receive glyph choice or roll -> socket glyph -> alter sphere/edge/center behavior -> create stronger Energy loops
```

### Glyph acquisition sources

Possible sources:

1. **Glyph Forge**: spend Energy to roll or reveal glyphs.
2. **Milestone rewards**: major sphere or Core Power milestones offer glyph choices.
3. **Bloom rewards**: occasional glyph fragments or rolls from milestone completions.
4. **Duplicate fusion**: repeated glyphs upgrade existing glyphs.
5. **Tech tree unlocks**: certain sphere paths unlock path-themed glyph pools.

### Recommended MVP acquisition model

Start with Glyph Forge plus milestone-choice rewards.

```txt
Glyph Forge:
Spend Energy to reveal 3 glyph choices.
Pick 1.
Each forge use increases the next forge cost.
Core Power unlocks better rarity pools.
```

This is more strategic and satisfying than a single blind roll, while preserving variable rewards.

### Escalating roll cost

Every glyph roll/forge should become more expensive.

Example formula:

```txt
nextGlyphForgeCost = baseCost * growthRate ^ totalGlyphForges
```

Suggested starting values:

```txt
baseCost = 100 Energy
growthRate = 1.35
```

Example cost curve:

| Forge # |    Cost |
| ------: | ------: |
|       1 |     100 |
|       2 |     135 |
|       3 |     182 |
|       4 |     246 |
|       5 |     332 |
|      10 |   2,107 |
|      20 | 42,000+ |

Numbers should be tuned against the Energy economy. The important shape is steep scaling.

### Glyph rarity

Potential rarity bands:

```txt
Common
Uncommon
Rare
Epic
Mythic / Legendary later
```

Rarity should affect complexity and synergy, not only raw magnitude.

Common glyphs:

```txt
+10% output.
+10% Bloom.
+5 Momentum on return.
```

Rare glyphs:

```txt
Convert high Momentum into bonus Sphere XP.
When this sphere Blooms, pulse Energy through all outgoing edges.
If this sphere has 3+ active connections, increase glyph trigger strength.
```

### Glyph choices instead of pure randomness

To avoid frustration, many glyph rewards should reveal multiple options.

```txt
Reveal 3 glyphs. Choose 1.
```

This supports strategy and gives the reward moment more agency.

### Glyph slots

Glyph slots are primarily sphere investment rewards.

A slot should feel like this sphere has become deep enough to hold more power, not like the player bought a generic global capacity upgrade. Core Power may unlock the glyph system itself, improve forge tiers, or occasionally grant a global/free slot later, but most sphere glyph slots should come from that sphere's own progression.

Recommended model:

```txt
Slot 1: unlocked by an early Sphere Point upgrade or first-sphere tutorial milestone.
Slot 2: requires Sphere Level 4 and a Sphere Point upgrade, possibly with an Energy activation cost.
Slot 3: requires Sphere Level 8 and deeper sphere investment, with a high Energy activation cost.
Special slots: unlocked by path/glyph/prestige systems.
Core Power: unlocks the Glyph Forge, higher rarity access, and rare global slot bonuses later.
```

Slots create long-term goals and make sphere identity matter. A sphere with multiple glyph slots should visibly read as a domain the user has repeatedly invested in.

### Glyph locations

Initial:

```txt
Sphere glyphs
```

Later:

```txt
Edge glyphs
Center glyphs
Ritual gems
```

## 8. Example player-facing loops

### Early game

```txt
Practice Music for 15 minutes.
Music gains 15 Sphere XP and reaches Level 2.
Music gains 1 Sphere Point.
Session produces Energy.
Spend Sphere Point on Bloom Rank 1.
Spend Energy at Center toward Core Power 2.
```

### Mid game

```txt
Fitness has high Momentum and Flow upgrades.
User routes some Fitness Energy through Music before Center.
Music has a glyph that rewards incoming Energy.
A milestone Bloom triggers extra flow through both edges.
Energy funds the next Glyph Forge.
```

### Glyph reward moment

```txt
User reaches Core Power 4.
New Glyph Forge tier unlocked.
Spend 2,400 Energy to reveal 3 glyphs.
Choose Resonant Bloom because it combos with the user's routed lattice.
Socket it into Music's second glyph slot.
Milestone completions now pulse Energy through connected spheres.
```

## 9. Current open questions

1. Should the first glyph reward happen at the first sphere slot unlock, at a Core Power milestone, or after the first major sphere milestone?
2. Should glyph forge cost scale globally, per rarity tier, or both?
3. Should Sphere Points alone buy identity nodes, or should stronger identity nodes also require Energy?
4. Should every sphere start with one glyph slot, or should the first slot be earned through early sphere investment?
5. How soon should manual routing percentages appear? Immediately, or after Core Power unlocks routing control?
6. Should Momentum-based XP bonuses be rare glyph/upgrades only, or part of a normal late-game branch?

## 10. Implementation implications

The domain model should move toward:

```ts
type GameState = {
  energy: number;
  corePowerLevel: number;
  coreUpgrades: CoreUpgradePurchase[];
  glyphForgeCount: number;
  glyphInventory: GlyphInstance[];
};
```

```ts
type Sphere = {
  id: string;
  name: string;
  xp: number;
  level: number;
  spherePointsEarned: number;
  spherePointsSpent: number;
  upgradePurchases: SphereUpgradePurchase[];
  momentum: number;
  charge: number;
  glyphSlots: GlyphSlot[];
};
```

```ts
type Connection = {
  id: string;
  fromSphereId: string;
  toSphereId: string;
  enabled: boolean;
  allocationPercent: number;
  level: number;
  mode: ConnectionMode;
};
```

Effect resolution should remain generic so sphere upgrades, Core Power upgrades, connection upgrades, and glyphs can all contribute modifiers.

---

## 11. North-star summary

The v0.2 core is:

```txt
Real time deepens individual spheres through Sphere XP.
Energy powers the whole lattice and buys scalable upgrades.
Core Power replaces global XP as the global upgrade spine.
Momentum makes consistency mechanically alive.
Connections become controllable parts of the game board.
Glyphs become escalating, variable rewards with strategic synergy.
```

This gives Snowball a clearer game economy while preserving the core emotional promise:

```txt
start easily, return safely, and let repeated attention become structure.
```
